# 爆款拆解功能专用 GStack 提示词模板

这份模板专门服务当前项目的“爆款拆解”和“提示词优化器”开发。

## 模板 1：先出产品和工程方案

```text
加载 gstack。先运行 gstack-office-hours，再运行 gstack-autoplan。

目标：
为当前项目新增“爆款拆解提示词优化器”能力。

业务背景：
- 当前项目是 AI 图片/视频生成 + 代理分销系统
- 已有视频拆解页 `/ai/video/analyze`
- 已有视频生成页 `/ai/video`
- 已有后端 PromptOptimizeService

我要的新能力：
- 用户输入一段粗糙需求
- 系统输出更适合短视频平台的结构化优化结果
- 输出必须包含：开头钩子、目标人群、卖点拆解、镜头拆解、文案策略、平台建议、中文成品 Prompt、英文成品 Prompt
- 优先适配豆包、阿里百炼、舞墨，也兼容通用国际模型

请输出：
1. 最小可上线方案
2. 前后端改动点
3. 接口设计
4. 输出 JSON 结构
5. 风险点和测试重点

不要直接编码。
```

## 模板 2：只审技术方案

```text
加载 gstack。使用 gstack-plan-eng-review。

请审查当前“爆款拆解提示词优化器”方案：
- 后端新增 `/ai/prompt/optimize/hot-video`
- 复用 ChatClient 调用 LLM
- 根据短视频业务输出结构化 JSON
- 前端新增一个独立工具页用于生成和复制 Prompt

重点检查：
1. 输出结构是否稳定
2. JSON 解析是否脆弱
3. 接口命名和模块边界是否合理
4. 是否需要 provider/model 解耦
5. 前端是否应该保存历史
```

## 模板 3：让它直接 review 这块代码

```text
加载 gstack。使用 gstack-review。

请 review 当前分支里“爆款拆解提示词优化器”相关改动，按生产风险排序输出问题。

重点关注：
1. Prompt 输出字段是否完整
2. JSON 解析失败时的降级逻辑
3. 前端异常态、空态、复制体验
4. 接口和页面命名是否清晰
5. 是否缺测试或文档
```

## 模板 4：用 QA 测这个工具页

```text
加载 gstack。使用 gstack-qa-only。

请测试 `http://localhost:8888/ai/prompt-optimizer`。

重点验证：
1. 空输入时按钮状态
2. 示例填充是否正确
3. 点击优化后的加载态
4. 返回结果后中文/英文 Prompt 是否可复制
5. 移动端布局是否可用

只输出问题报告，不要修改代码。
```
